A vertical map of the modern web stack

From markup to machine intelligence

Six layers, one continuous signal. Scroll down to follow a single request as it moves from the page you see, through the server that built it, to the model that reasons about it.

SCROLL
LAYER 01

HTML

// the skeleton — what exists on the page

HTML gives the browser its nouns. Every heading, button, and image is a tagged piece of meaning before any style or behavior touches it.

Semantic tags like <nav>, <article>, and <button> aren't cosmetic — screen readers, search engines, and your own CSS selectors all key off this structure.

↓ structure is in place. CSS arrives to give it shape and feeling.
LAYER 02

CSS

// the surface — how it looks and responds

CSS turns structure into an experience. The cascade resolves competing rules; specificity decides who wins.

Flexbox and Grid handle layout, media queries handle different screens, and transitions add the small motions that make an interface feel alive rather than static.

↓ a styled, static page exists. To get live data, it calls an API.
LAYER 03

API

// the contract — how the frontend asks, and the backend answers

An API is a promise about shape. The browser sends a request — usually GET, POST, PUT, or DELETE — to a URL, and gets back data, almost always JSON.

REST keeps that contract simple and predictable; this is the seam where frontend and backend stop needing to know anything about each other's internals.

↓ the contract is defined. React builds the UI that consumes it.
LAYER 04

React

// the composition — UI as components and state

React turns the page into a tree of components — small functions that take data in (props) and return what should be on screen.

useState and useEffect let a component remember things and react to change; when data from the API updates state, React re-renders only what actually needs to change.

↓ the UI is composed. Node is what runs the server side of that API call.
LAYER 05

Node.js

// the runtime — where requests get handled

Node runs JavaScript outside the browser, which is what lets a server written in the same language as the frontend handle incoming requests.

Its event loop processes I/O — database queries, file reads, outbound API calls — without blocking, so one process can serve many requests concurrently. Frameworks like Express map routes to logic.

↓ the request reaches the server. Increasingly, that server calls an AI engine.
LAYER 06

AI Engine Integration

// the reasoning — where the response gets intelligence

From the server, a request to a model API — sending a prompt and context, receiving generated text, a classification, or a structured result back as JSON, same as any other API call.

The pattern repeats what every layer below already established: the server holds the API key and orchestration logic, the model does the reasoning, and the response flows back down through Node → API → React → the page you started on.

↑ the response travels back up every layer to render on the page you saw first.

One request, six layers, no magic

Every layer above is just a more specific case of the one below it solving a real constraint: show something, style it, fetch data, compose it, run it somewhere, make it smarter.
